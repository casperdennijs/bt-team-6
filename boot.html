2. Custom fonts

De tekst heeft een belangrijke functie voor de gebruiker, omdat dit vaak de content is waarom de gebruiker de pagina bezoekt. Custom fonts kunnen bijdragen aan de visuele stijl van een website, maar het is belangrijk om de complicaties te overwegen van custom fonts.

  1. Leesbaarheid

    2. Sommige custom fonts zijn moeilijk om te lezen. Het is ook belangrijk om de fonts te testen op verschillende apparaten, zodat deze leesbaar is op verschillende schermen.

  2. Grootte

    3. Grote bestanden duren langer om te laden en dit kan ervoor zorgen dat de pagina langer gaat laden. Dit kan zorgen voor frustratie bij de gebruiker en dat de gebruiker de pagina verlaat. Het is daarom belangrijk om je font te optimizen, zodat deze sneller laden.

  3. Fallback fonts

    4. Het is belangrijk om fallback fonts te gebruiken bij het gebruik van custom fonts. Als de custom font niet kan worden geladen, dan wordt de fallback font geladen. Het is een "good practice" om web-safe fonts te gebruiken als fallback fonts, omdat deze vaak beschikbaar zijn op verschillende apparatuur.

  4. Metadata

    5. Je moet ervoor zorgen dat de tekst accessible is voor gebruikers met een screenreader, dit kan je bijvoorbeeld doen door metadata te gebruiken voor custom fonts, zoals font name, font designer, font license, font style, font version, font format, font copyright, font description en font usage.

3. Kleur uitzetten & kleurenblindheid instellen

Een van de meeste belangrijke manieren van communicatie in web design is kleur. Het is handig om doormiddel van kleur iets te communiceren naar mensen, maar het is een uitdaging voor de 300 miljoen mensen wereldwijd met een vorm van kleurenblindheid.

Soorten kleurenblindheid

  1. Rood-groen kleurenblindheid

    2. Deze komt het meest voor en zorgt ervoor dat de persoon rood en groen door elkaar heen haalt.

  2. Blauw-geel kleurenblindheid

    3. Het komt minder vaak voor maar dit zorgt ervoor dat de persoon blauw en geel door elkaar haalt.

  3. Monochromasie

    4. Dit is de meest zeldzame vorm van kleurenblindheid en zorgt ervaar dat de persoon helemaal geen kleur ziet.

Best practices om een kleurenblindheid vriendelijke website te maken

  1. Vertrouw niet teveel op kleur voor communicatie
  2. Maak gebruik van icons en symbolen
  3. Gebruik textuur en patronen
  4. Gebruik labels
  5. Ga voor bold
  6. Maak goed gebruik van contrast
  7. Vermijd bepaalde kleuren combinaties
  8. Omarm minimalisme

5. Breedband internet

Tijdens het onderzoek heb ik gebruik gemaakt van de ingebouwde Network Throttling op de preset “Slow 3G”. Daarbij heb ik gekozen voor de website NOS.nl, omdat hier veel content ingeladen moet worden.

De NOS.nl website laad bij een lage internetsnelheid nog redelijk snel hun eerste content in. Zo krijg je vrij snel al een soort skeleton te zien en zie je teksten van nieuws artikelen. Wat wel vreemd is, is dat de teksten eerder getoond worden dan de titels wat soms een klein beetje verwarrend kan zijn omdat je dan context mist.

Verdere bevindingen die ik heb opgedaan tijdens het onderzoeken zijn als volgt:

  • Nieuws teksten gaan vrij snel en je krijgt ook meteen een indruk van de layout doordat er skeletons gemaakt zijn tijdens het laden
  • Teksten komen eerder dan de titels, dit kan soms een beetje ervoor zorgen dat je wat context mist als je de tekst meteen gaat lezen
  • Op mobiel werken het menu niet meteen, dit komt omdat de dropdowns gemaakt zijn met JavaScript. Dit zou ook in CSS gemaakt kunnen worden waarbij je geen JavaScript nodig hebt
  • Afbeeldingen worden geleidelijk geladen, je kan de progressie ervan heel goed zien. Het begint bovenaan met laden en vult zich langzaam naar beneden
  • De desktop versie lijkt nog iets sneller in te laden dan de mobiele versie. Je krijgt iets eerder content getoond.
  • Vroegtijdig het laden laten stoppen kan ervoor zorgen dat content zoals plaatjes helemaal niet ingeladen worden. Ook niet de delen die daarvoor wel al deels kon zien.

Wat voor impact kan dit hebben voor de gebruiker:

Iedere persoon is natuurlijk anders en de één heeft meer geduld dan de andere. Het is dan dus heel belangrijk dat ondanks hun mindere omstandigheden niet te lang moeten hoeven wachten

6. Javascript

JavaScript is niet altijd beschikbaar en het is niet de schuld van de gebruiker

Ongeveer 1% van alle gebruikers van het web zetten JavaScript bewust uit. Veel developers denken dat dit betekent dat ze deze mensen buiten kunnen sluiten. Ze vergeten dat 1% nog steeds een groot getal is, vooral relatief gezien. Voor een site als bijvoorbeeld BuzzFeed is 1% 13 miljoen requests per maand.

Een paar redenen waarom gebruikers een website ervaren alsof er geen JavaScript aanwezig is:

1. De browser herkent de JavaScript code niet

Als je JavaScript-bestand methoden of eigenschappen bevat die de browser niet begrijpt, wordt het script niet uitgevoerd.

2. Browserextensies kunnen JavaScript breken

Browser-plug-ins gebruiken JavaScript om webpagina's te wijzigen en functionaliteit toe te voegen. Maar dit loopt het risico te breken, afhankelijk van hoe goed het JavaScript van de site en het JavaScript van de plug-in is geschreven.

3. Sommige browsers schakelen JavaScript uit bij trage verbindingen

Chrome op Android schakelt JavaScript bijvoorbeeld automatisch uit bij langzame verbindingen om de laadtijden van pagina's te versnellen.

4. JavaScript kan mogelijk niet worden gedownload

Zoals bij elke HTTP-aanroep, kan het verzoek om het JavaScript van je site niet worden geladen. Dit kan gebeuren in een trein omdat de verbinding stopt voordat het script is gedownload.

5. Bedrijfsfirewalls en mobiele operators blokkeren JavaScript

Sky Broadband heeft bijvoorbeeld ooit per ongeluk JQuery geblokkeerd, waardoor veel sites kapot gingen.

6. Sommige browsers voeren geen JavaScript uit

Sky Broadband heeft bijvoorbeeld ooit per ongeluk JQuery geblokkeerd, waardoor veel sites kapot gingen.

Mijn onderzoek

Hoe meer je site afhankelijk is van JavaScript, hoe slechter de ervaring waarschijnlijk zal zijn wanneer JavaScript niet beschikbaar is. Als je te maken hebt met tekst, afbeeldingen, video's en formulieren, geef de HTML dan weer op de server. Geen JavaScript nodig, dit is robuuster en houdt meer rekening met performance.

Om uit te testen hoe erg missende JavaScript een site beperkt, heb ik de JavaScript uitgezet op de site van bol.com.

Vergeleken met veel andere sites houdt bol.com erg goed rekening met mensen die geen JavaScript (kunnen) gebruiken. Bijna de hele site werkt, inclusief het bestellen van spullen en het beheren van je account, wat tegen mijn verwachting in gaat.

Dingen die niet werken is wanneer je het menu openklapt, en in dit menu een sub-menu openklapt. Ook werkt de chatbot niet. Sommige foto's laden ook niet in.

7. Cookies en localStorage

Cookies zijn kleine tekstbestanden die door een website worden geplaatst op de computer van een gebruiker wanneer deze de website bezoekt. Deze cookies bevatten informatie over bijvoorbeeld de items in een winkelmandje of een gebruikersnaam en wachtwoord om automatisch in te loggen. Deze informatie staat dan paraat zodra de gebruiker de website weer bezoekt. Het verschil met local storage is dat local storage wordt opgeslagen in de cache van de browser. Zodra local storage en cookies uit worden gezet kunnen dit soort functionaliteiten wegvallen. Local storage kan in de instellingen van de meeste moderne browsers uitgezet worden. Cookies moet de gebruiker zelf weigeren op een website.

De cookiewet

De cookiewet verplicht websites om toestemming te vragen aan hun gebruikers om cookies toe te passen op hun websites. Dit in verband met de privacy van de eindgebruiker. Dit geeft gebruikers de mogelijkheid om te bepalen welke informatie ze willen delen met websites en het voorkomt dat websites ongewenst gegevens verzamelen zonder toestemming. De cookiewet is bedoeld om de privacy van gebruikers te beschermen en om ervoor te zorgen dat websites transparanter zijn over hun gebruik van cookies.

De wet brengt echter nog een groot probleem met zich mee omdat veel gebruikers de cookiemeldingen op websites negeren en gewoon doorgaan met browsen. Dit betekent dat gebruikers nog steeds worden gevolgd en dat hun gegevens worden verzameld, ondanks de wet. Een andere oplossing zou zijn om de inhoud die mag opgeslagen worden in cookies te reguleren.

8. Content blockers en ad blockers

Advertenties op het web. Iedereen heeft er een hekel aan, maar gelukkig zijn er middelen om hun impact op het web te limiteren. Een voorbeeld hiervan is AdBlockPlus. Met behulp van deze browser extensie is het mogelijk om advertenties op het web te blokkeren. Door deze extensie toe te voegen aan je browser is het mogelijk om de impact van advertenties op het web te zien.

Wat blokkeren adblockers precies?

Adblockers zijn in staat om alle vormen van advertenties op het web te blokkeren. Denk hierbij aan de volgende vormen.

  • Banner advertenties
  • Video advertenties
  • Pop up advertenties
  • Reguliere advertenties
  • Tracking codes die informatie verlenen aan marketing teams

Welke impact heeft het gebruik van een adblocker

Het meest logische gevolg van het gebruik van een adblocker dat gebruikers niet langer naar irritante advertenties hoeven te kijken, maar het heeft ook andere gevolgen. Denk hierbij aan de laadtijd van een pagina. Zodra de advertenties worden geblokkeerd zal er minder data ingeladen moeten worden. Hierdoor zal de totale laadtijd van de pagina afnemen. Dit is bijvoorbeeld zichtbaar op de website van de telegraaf of het algemeen dagblad. Hier zie je op de homepagina duidelijk content verspringen zodra de advertenties als laatst worden ingeladen.

Een ander nadelig gevolg van het gebruik van adblockers is dat sommige pagina’s niet zichtbaar zijn voor gebruikers met een adblocker. Deze websites gebruiken een script om de adblocker te herkennen en tonen vervolgens een pop up om de pagina te verbergen. Een voorbeeld hiervan is de website https://iconmonstr.com/.

Uit onderzoek blijkt dat 52% van de Nederlandse jongeren op hun laptop een adblocker gebruikt. Dit percentage ligt op mobiel lager, namelijk 19%. Een gevolg hiervan is dat websites die hun inkomsten grotendeels op advertenties baseren veel omzet mislopen. Mocht het aantal adblockers toenemen kan dit ertoe leiden dat deze websites niet langer zullen bestaan.

Wat is het verschil met een content blocker?

Een content blocker werkt soortgelijk als een adblocker. Beide geven de gebruiker de optie om content te verschuilen op het web. Het verschil met een reguliere adblocker is dat de gebruiker met een content blocker ook andere content op websites kan verschuilen naast alleen advertenties. Denk hierbij aan dingen zoals social buttons, comments onder een artikel of analytics. Wat hiermee wordt bereikt is een nog kortere laadtijden.
